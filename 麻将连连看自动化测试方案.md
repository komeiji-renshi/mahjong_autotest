我有一个麻将连连看的游戏，我想做一个自动化跑测脚本，通过adb连接手机，采用图像识别的方式让这个游戏的关卡可以全自动化运行，你觉得有什么难点不？



很多游戏叫 **“麻将连连看 / Mahjong Solitaire / 上海麻将”** 这一类。

它的结构和普通棋盘连连看完全不同，所以我前面说的 **“固定网格切格子”方案就不太适用了**。

因为它有三个关键特征：

1️⃣ **牌是堆叠的（有层级 z-axis）**
 2️⃣ **上层牌会遮挡下层牌**
 3️⃣ **牌的位置不是严格网格，有半格偏移**

例如典型布局：

```
        [A]
    [B] [C] [D]
[E] [F] [G] [H] [I]
```

并且可能是三层：

```
   layer3
      A

   layer2
   B   C

   layer1
D  E  F  G
```

这类游戏的自动化难度 **确实比普通连连看高很多**。

不过还是能做，只是识别思路要换一套。

------

# 一、这种麻将连连看的核心难点

相比普通连连看，多了三个关键难点：

### 1️⃣ 不再是规则网格

普通连连看：

```
8 × 10 grid
```

麻将连连看：

```
坐标是自由摆放
```

例如：

```
tile1: (500,300)
tile2: (540,280)
tile3: (580,260)
```

所以你不能简单用：

```
row col
```

去表示棋盘。

------

### 2️⃣ 有遮挡关系

例如：

```
上层牌
   A

下层牌
 B   C
```

A 会挡住 B 和 C 的一部分。

你必须判断：

- 哪些牌 **可点击**
- 哪些牌 **被压住**

通常规则是：

> 一张牌如果 **没有被上方压住** 且 **左或右至少一侧空** 才能点击。

------

### 3️⃣ 偏移布局

麻将连连看经常这样排：

```
   [A]
[B]   [C]
   [D]
```

也就是说：

```
x 轴和 y 轴不严格对齐
```

所以不能用简单的 `(row, col)` 切格子。

------

# 二、麻将连连看的推荐识别思路

核心思想是：

> **先识别所有牌的位置 → 再判断层级关系 → 再识别牌类型**

整体流程是：

```
截图
 ↓
识别所有牌位置
 ↓
识别每张牌的图案类型
 ↓
判断遮挡关系
 ↓
筛选可点击牌
 ↓
找相同牌对
 ↓
点击
 ↓
循环
```

------

# 三、关键步骤 1：识别所有牌的位置

这一步是整个系统最关键的。

常见方法有三种。

------

## 方法1：模板匹配找牌框（最推荐）

麻将牌通常有固定边框，例如：

```
┌──────┐
│ 🀄   │
└──────┘
```

你可以用：

```
牌框模板
```

去做 **find_all**。

例如：

```
tiles = find_all(Template("tile_frame.png", threshold=0.8))
```

得到：

```
[(x1,y1), (x2,y2), (x3,y3)...]
```

每个就是一张牌的位置。

优点：

- 实现简单
- 稳定性高
- 不受图案影响

缺点：

- 要有清晰牌框

------

## 方法2：OpenCV 轮廓检测

如果牌框比较明显，可以：

1️⃣ 灰度化
 2️⃣ Canny 边缘检测
 3️⃣ 找矩形轮廓

筛选：

```
宽高接近麻将牌比例
```

就能得到所有牌。

------

## 方法3：目标检测（最强但最复杂）

用 YOLO / CNN 训练：

```
tile detection
```

模型输出：

```
bounding boxes
```

优点：

- 抗遮挡强
- 通用性好

缺点：

- 训练成本高
- 对测试项目通常没必要

------

# 四、关键步骤 2：识别牌的图案类型

麻将连连看通常有 **30~40 种牌**。

你可以两种方式做。

------

## 方案A：模板匹配

每种牌准备一个模板：

```
bamboo_1
bamboo_2
dragon_red
...
```

然后对每个 tile：

```
match template
```

这种方法：

优点：

- 精度高
- 实现简单

缺点：

- 要准备模板

------

## 方案B：自动聚类（我更推荐）

和普通连连看一样：

只要识别

```
哪些牌长得一样
```

不需要知道名字。

算法：

```
for tile in tiles:
    if similarity(tile, existing_class):
         assign class
    else
         new class
```

最后你得到：

```
tile1 → class 3
tile2 → class 5
tile3 → class 3
tile4 → class 2
```

就可以找：

```
class 3 pair
```

------

# 五、关键步骤 3：判断遮挡关系（重点）

麻将连连看的关键规则：

> 如果一张牌被上方牌覆盖，就不能点击。

例如：

```
   A
 B   C
```

A 会挡住 B 和 C。

判断方法其实不难：

只需要用 **bounding box overlap**。

例如：

```
tile_A: (x,y,w,h)
tile_B: (x2,y2,w2,h2)
```

如果：

```
A.y < B.y
并且
A.x 在 B.x ± 半个牌宽
```

就说明：

```
A 在 B 上面
```

然后建立一个关系：

```
B.covered_by = [A]
```

如果 `covered_by` 不为空：

```
B 不可点击
```

------

# 六、关键步骤 4：判断左右是否被挡

麻将规则一般是：

```
左 或 右 至少一侧没有牌
```

才能点击。

判断方法：

检查：

```
tile 左侧一定范围是否有牌
tile 右侧一定范围是否有牌
```

如果：

```
left_blocked AND right_blocked
```

则：

```
tile 不可点击
```

------

# 七、最终数据结构

最后你会得到一堆 tile 对象：

```
class Tile:
    id
    x
    y
    w
    h
    class_id
    covered_by
```

例如：

```
Tile1 class=3 free
Tile2 class=3 free
Tile3 class=5 blocked
Tile4 class=2 free
```

------

# 八、求解逻辑就很简单

筛选：

```
free tiles
```

按 class 分组：

```
class 3 → [tile1, tile2]
class 5 → [tile3]
class 2 → [tile4]
```

如果：

```
len(group) >= 2
```

就点击：

```
tile1
tile2
```

------

# 九、完整流程

自动化循环：

```
while True:

    screenshot

    detect tiles

    classify tiles

    compute blocking

    free_tiles = filter(clickable)

    pair = find_same_pair(free_tiles)

    if pair:
        tap(pair[0])
        tap(pair[1])
    else:
        break
```

如果普通连连看难度是：

```

麻将连连看通常是：

```

 2️⃣ 偏移布局

 4️⃣ 动态动画

但它也有一个优点：

**规则比连连看简单。**

因为它不需要：

```

```

```

# 十一、一个非常重要的实践经验

很多人做这种自动化会犯一个错误：

**每一帧都重新检测所有牌。**

其实更好的做法是：

```
初始识别全部牌
```

然后每次消除：

```
只更新状态
```

因为牌的位置通常不会变。

这样效率会高很多。

## 1. 设备控制层

文件建议：

### 负责什么

- ADB 连接设备
- 检查设备在线状态
- 截图
- 点击
- 滑动
- 启动游戏
- 回到桌面
- 重启游戏

### 关键接口建议

```
class AdbController:

### 重点

这里要做到**绝对稳定**。
 因为很多跑测失败，根本不是识别算法问题，而是：

- adb 截图挂了
- tap 没点上
- 设备断连

## 2. 图像预处理层

文件建议：

- `vision/preprocess.py`

### 负责什么

- 原图裁剪
- 缩放
- 灰度化
- 去噪

### 关键接口

def preprocess_screen(img: np.ndarray) -> np.ndarray: ...

## 3. 牌检测层

文件建议：

- `vision/tile_detector.py`

这是整个项目最重要的一层之一。

### 负责什么

从截图中找出**当前所有可见牌**的位置。

输出的不是分类结果，而是一堆牌框：

```
[
    (x, y, w, h),
    (x, y, w, h),
    ...
]
```

### 推荐实现路线

我建议优先做 **双通道检测**：

### 方案A：模板匹配检测牌框

适合：

- 牌外框样式固定
- UI 比较稳定

### 方案B：轮廓检测兜底

适合：

- 模板匹配漏检
- 想提升泛化

### 接口建议

```
class TileDetector:
    def detect(self, img: np.ndarray) -> list["TileBox"]:
        ...
```

### TileBox 数据结构建议

```
from dataclasses import dataclass

@dataclass
class TileBox:
    x: int
    y: int
    w: int
    h: int
    confidence: float
```

### 检测层核心难点

你这个游戏不是规则格子，所以这层必须解决：

- 半格偏移
- 多层堆叠
- 牌间部分遮挡
- 有些牌只露出一部分

### 实战建议

第一版不要追求“连被挡住 70% 的牌也能识别”。
 只检测**当前可见主体足够大的牌**就够了，因为你主要要操作的是**可点击牌**。

------

## 4. 牌分类层

文件建议：

- `vision/tile_classifier.py`

### 负责什么

给每个检测到的牌，分配一个 `class_id`。
 不一定非要知道它是“3万”还是“白板”，只要知道“长得一样”即可。

### 推荐两阶段分类

#### 阶段1：特征提取

对每张牌提取可比较的图像特征。

比如：

- 灰度模板
- 感知哈希
- ORB 特征
- SSIM 相似度输入
- 中心图 patch

#### 阶段2：聚类归类

把相似的牌归到同一个类。

### 接口建议

```
class TileClassifier:
    def classify(self, tile_imgs: list[np.ndarray]) -> list[int]:
        ...
```

或者：

```
class TileClassifier:
    def assign_class(self, tile_img: np.ndarray) -> int:
        ...
```

### 推荐策略

第一版用**在线聚类**最合适：

```
classes = []
for tile in detected_tiles:
    for cls in classes:
        if similarity(tile, cls.prototype) > threshold:
            assign cls_id
            break
    else:
        create new cls_id
```

### 为什么我推荐在线聚类

因为测试项目里你真正关心的是：

- 哪两张牌相同

而不是：

- 它具体是什么麻将名字

这样就省掉大量模板维护成本。

------

## 5. 局面建模层

文件建议：

- `model/tile.py`
- `model/board_state.py`
- `vision/overlap_analyzer.py`

这一层负责把“检测结果”变成“规则可计算的游戏状态”。

------

### `Tile` 对象建议

```
from dataclasses import dataclass, field

@dataclass
class Tile:
    id: int
    bbox: tuple[int, int, int, int]
    center: tuple[int, int]
    class_id: int | None = None
    visible_ratio: float = 1.0
    covered_by: list[int] = field(default_factory=list)
    left_blocked: bool = False
    right_blocked: bool = False
    clickable: bool = False
```

------

### `BoardState` 对象建议

```
@dataclass
class BoardState:
    tiles: list[Tile]
    timestamp: float
    screenshot_path: str | None = None

    def clickable_tiles(self) -> list[Tile]:
        return [t for t in self.tiles if t.clickable]

    def group_by_class(self) -> dict[int, list[Tile]]:
        ...
```

------

### 这一层做什么

- 计算谁挡住谁
- 计算左右是否被挡
- 计算哪些牌当前可点击
- 维护当前局面的整体状态

------

### 遮挡判断核心思路

#### 上方遮挡

若牌 A 和牌 B 满足：

- A 的中心在 B 的上方
- A 与 B 的 bbox 有特定比例重叠
- A 的位置符合“上层偏移关系”

那么可认为：

```
B.covered_by.append(A.id)
```

#### 左右阻挡

看某张牌左侧/右侧相邻区域是否存在其他牌中心或牌框侵入：

```
tile.left_blocked = has_neighbor_left(tile)
tile.right_blocked = has_neighbor_right(tile)
```

#### 可点击规则

通常可写成：

```
tile.clickable = (len(tile.covered_by) == 0) and (not (tile.left_blocked and tile.right_blocked))
```

这就是麻将连连看的常见判定。

------

## 6. 求解决策层

文件建议：

- `solver/accessibility.py`
- `solver/pair_matcher.py`
- `solver/strategy.py`
- `solver/planner.py`

### 负责什么

根据当前 `BoardState` 找到下一步该点哪两张牌。

------

### 最基础策略

1. 找所有 `clickable tiles`
2. 按 `class_id` 分组
3. 找出组内数量 >= 2 的类
4. 输出一对牌

### 接口建议

```
class PairMatcher:
    def find_pairs(self, board: BoardState) -> list[tuple[Tile, Tile]]:
        ...
class Strategy:
    def choose_action(self, pairs: list[tuple[Tile, Tile]], board: BoardState) -> tuple[Tile, Tile] | None:
        ...
```

------

### 为什么还要单独有 strategy

因为“找到一对可消牌”和“选哪一对最好”不是一回事。

#### 第一版可以很简单

- 找到第一对就点

#### 后面可以进阶

- 优先点最上层牌
- 优先点能释放更多牌的对子
- 优先减少遮挡链
- 避免点完后进入死局

------

# 六、完整运行流程

我给你一版可落地的完整主流程。

## 单局主循环

```
while not finished:
    img = adb.screencap()

    game_state = state_recognizer.recognize(img)

    if game_state == "popup":
        recovery.handle_popup(img)
        continue

    if game_state == "win":
        result.mark_success()
        break

    if game_state == "fail":
        result.mark_fail()
        break

    tile_boxes = detector.detect(img)
    if not tile_boxes:
        recovery.handle_no_tiles(img)
        continue

    tile_imgs = [crop(img, tb) for tb in tile_boxes]
    class_ids = classifier.classify(tile_imgs)

    tiles = build_tiles(tile_boxes, class_ids)
    board = analyzer.build_board_state(tiles)

    pairs = matcher.find_pairs(board)
    if not pairs:
        recovery.handle_no_pairs(board, img)
        continue

    action = strategy.choose_action(pairs, board)
    executor.tap_pair(action)

    wait_animation_finish()
```

for level_index in range(max_levels):

# 八、异常恢复机制怎么做

这一块特别重要，我单独展开。

------

## 1. 截图异常恢复

现象：

- 截图为空
- 截图花屏
- adb timeout

处理：

- 重试 2~3 次
- 失败则重连 adb
- 再失败则重启 app

------

## 2. 无法识别牌

现象：

- `tile_boxes == []`
- 或数量异常过少

处理：

- 再截一次图
- 等 0.5 秒
- 检查是不是动画态/过渡态
- 检查是否进入结算页/弹窗页

------

## 3. 找不到可消对子

可能原因：

- 识别错了
- 实际进入死局
- 牌未完全稳定
- 有道具/洗牌按钮

处理顺序建议：

1. 再识别一次
2. 等待动画完成
3. 找洗牌按钮
4. 点击提示或洗牌
5. 仍失败则判定局面卡死

------

## 4. 点击无效

现象：

- 点完后局面没变化

处理：

- 重新检查点击点位是否偏
- 对点击加入随机微偏移
- 点击间隔适当拉开
- 达到阈值后重新识别局面

------

## 5. 长时间无进展

定义一个 watchdog：

例如：

- 15 秒没有牌数变化
- 10 次操作无消除结果

则触发恢复：

- 打开暂停
- 退出重开
- 或记录失败并进入下一关

# 十六、一个简化版核心伪代码

你可以把整个项目脑子里先理解成这样：

```
def run_one_step():
    img = adb.screencap()

    if ui.is_win(img):
        return "win"
    if ui.is_fail(img):
        return "fail"
    if ui.has_popup(img):
        ui.close_popup(img)
        return "popup"

    tile_boxes = detector.detect(img)
    tiles = []

    for i, box in enumerate(tile_boxes):
        tile_img = crop(img, box)
        cls_id = classifier.assign_class(tile_img)
        tiles.append(Tile(
            id=i,
            bbox=box,
            center=get_center(box),
            class_id=cls_id,
        ))

    board = analyzer.build(tiles)

    pairs = matcher.find_pairs(board)
    if not pairs:
        return "no_pair"

    pair = strategy.choose_action(pairs, board)
    adb.tap(*pair[0].center)
    adb.tap(*pair[1].center)

    return "acted"
```
